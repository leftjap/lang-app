## 8-A. 복습 큐 (활성 문장만)

`japanese-data.json` → `reviewQueue` 배열에서 관리한다.

### 주기 규칙
- 학습 당일 = Day 0
- 1일 후 → 3일 후 → 7일 후 → 21일 후 → 60일 후
- O(통과): 다음 주기로 이동
- △(부분 통과): 통과 처리하되 현재 주기와 다음 주기의 중간 날짜에 추가 복습 1회 삽입
- X(리셋): Day 0으로 리셋
- 60일 통과: reviewQueue에서 제거

### 용량 관리
- reviewQueue 길이가 80을 넘으면 신규 학습을 3개로 제한한다.
- 100을 넘으면 졸업 후보를 학습자에게 제시한다.

---

## 8-B. 한자 학습 큐

`japanese-data.json` → `kanjiQueue` 배열에서 관리한다.

### 주기 규칙
§8-A와 동일 (1일 → 3일 → 7일 → 21일 → 60일).

### 등록 기준
Step 2에서 새 문장에 포함된 한자 중 학습자가 처음 접하는 것만 등록한다.

### 용량 관리
- kanjiQueue 길이가 60을 넘으면 신규 한자 도입을 세션당 2개로 제한한다.
- 80을 넘으면 졸업 후보를 학습자에게 제시한다.

## 10. 학습 카테고리 (진행 상태 포함)

### 카테고리 목록

| # | 카테고리 | 설명 | 상태 |
|---|---|---|---|
| A | 기본 반응 표현 | そうだ, すごい, やばい, まじで, なるほど, うそ 등 | 대기 |
| B | 기본 문장 구조 | ～です/～ます, ～たい, ～ている, ～ない | 대기 |
| C | 의지/감정 표현 | 俺がやる, 絶対に負けない, 守りたい, 許さない 등 | 대기 |
| D | 조사 마스터 | は/が/を/に/で/から/まで | 대기 |
| E | 일상 대화 | 인사, 안부, 감사, 사과, 부탁, 거절 | 대기 |
| F | 동사 활용 | ます→て→ない→た→ば/たら | 대기 |
| G | 여행 실전 | 식당, 편의점, 역, 호텔, 돌발 상황 | 대기 |
| H | 애니 구어체 심화 | 축약, 종조사, 캐릭터별 말투 | 대기 |

### 진행 규칙
- 카테고리 A, B는 병행 진행한다.
- 한 카테고리에서 핵심 표현 20개 이상이 7일 주기(3회차 복습)를 통과하면 완료 여부를 판단한다.
- 완료된 카테고리의 문장도 60일 복습 주기까지 계속 복습 대상에 포함한다.

## 14. AI 행동 규칙

### 학습 생성 규칙
- §8-A 복습 큐 전체 행 수가 80을 넘으면 신규 학습을 3개로 제한한다.
- 복습 대상(문장+한자 합산)이 15개 이상이면 신규 학습을 3개로 줄인다.
- 복습 대상이 5개 미만이면 신규 학습을 8개까지 늘린다.
- 한 세션에 새 한자는 최대 5개까지만 도입한다.
- 복습 대상이 10개 이상이면 대화를 1개로 줄이고 신규 표현을 1개로 제한한다.

### 난이도 조절 규칙
- 복습 성공률이 최근 3세션 연속 90% 이상이면 다음 카테고리를 병행 시작한다.
- 복습 성공률이 최근 3세션 연속 50% 미만이면 신규 학습을 중단하고 복습에 집중한다.
- 작문에서 같은 실수가 3회 반복되면 해당 문법/패턴을 별도 미니 레슨으로 진행한다.

### 한자 도입 규칙
- 카테고리 A(기본 반응 표현)에서는 한자를 적극 도입하지 않는다.
- 카테고리 B부터 문장에 포함된 기초 한자를 자연스럽게 도입한다.
- 새 한자를 소개할 때 반드시 한국어 한자음을 함께 알려준다.

## 15. 문서 업데이트 작업지시서 (VS Agent / Haiku용)

### 역할 분담

| 역할 | 담당 | 원칙 |
|---|---|---|
| 학습 진행 + 판단 | Opus 4.6 | 복습 판정, 신규 생성, 약점 분석, 난이도 조절 |
| 작업지시서 생성 | Opus 4.6 | 세션 종료 시 판단 결과를 확정값으로 출력 |
| 파일 편집 | Haiku (VS Agent) | 작업지시서를 읽고 JSON을 편집 |

### 트리거

학습자가 대화 중 **"작업지시서"**라고 말하면, Opus는 즉시 해당 세션의 변경 사항을 반영한 작업지시서를 출력한다.

## 16. 이 문서의 관리

이 문서의 목적은 AI가 매 세션 읽고 행동 규칙과 데이터 스키마를 파악하는 것이다. 학습 상태 데이터는 `japanese-data.json`에서 관리한다.

**핵심 원칙:** JA-core.md가 행동 규칙을, JA-method.md가 방법론을, japanese-data.json이 학습 상태를 각각 담당한다.
