## 6. 학습자 약점 (AI가 업데이트)

학습 과정에서 발견된 약점. `english-data.json` → `weaknesses` 배열에서 관리한다.

상태: active / improving / resolved

### 예상 약점 (프로필 기반 추정, 학습 후 검증)
- 연음/축약 인식 (gonna, wanna, didja 등)
- 강세 없는 단어 청취 (기능어: a, the, to, of 등이 삼켜짐)
- 돌발 상황에서 문장 구성 (아는 단어는 있는데 조립이 안 됨)
- R/L, V/B, TH, F 발음 구분 (듣기에서도 영향)

## 7. 학습자 강점 (AI가 업데이트)

`english-data.json` → `strengths` 배열에서 관리한다.

- 학습 동기 높음 (자발적, 구체적 목표 있음)
- 체계적 접근 선호 (코딩 워크플로우 문서 운영 경험)

`english-data.json` → `sessionLogs` 배열에서 관리한다.

### 로그 항목 스키마 (참조용)

각 세션 로그는 아래 구조를 따른다:
```json
{
  "day": 1,
  "date": "2026-03-15",
  "category": "A",
  "durationMin": 25,
  "newSentenceIds": ["D1-1", "D1-2"],
  "reviewResults": [
    {"id": "D1-1", "result": "O"}
  ],
  "dictation": {"type": "dictation", "notes": "gonna 놓침"},
  "writing": {
    "situation": "카페에서 주문 변경",
    "original": "I want change my order",
    "corrected": "I'd like to change my order",
    "weaknessFound": "would like 패턴 미사용"
  },
  "voice": null,
  "aiNotes": "축약형 인식 약함, 다음 세션에 반영"
}
```

<!-- §8A-REVIEW -->
## 8-A. 복습 큐 (활성 문장만)

`english-data.json` → `reviewQueue` 배열에서 관리한다.

### 항목 스키마 (참조용)

```json
{
  "id": "D1-1",
  "sentence": "Could you say that again?",
  "keyExpression": "say that again",
  "category": "A",
  "learnedDate": "2026-03-15",
  "currentInterval": 1,
  "nextReview": "2026-03-16",
  "consecutivePasses": 0,
  "lastResult": null
}
```

### 주기 규칙
- 학습 당일 = Day 0
- 1일 후 → 3일 후 → 7일 후 → 21일 후 → 60일 후
- O(통과): 다음 주기로 이동
- △(부분 통과): 통과 처리하되 현재 주기와 다음 주기의 중간 날짜에 추가 복습 1회 삽입
- X(리셋): Day 0으로 리셋, consecutivePasses 0으로 초기화
- 60일 통과: reviewQueue에서 제거, stats.reviewCompleted60d +1

### 용량 관리
- reviewQueue 길이가 80을 넘으면 신규 학습을 3개로 제한한다.
- 100을 넘으면 졸업 후보(nextReview가 가장 먼 것 중 consecutivePasses ≥ 3)를 학습자에게 제시한다.

## 10. 학습 카테고리 (진행 상태 포함)

### 카테고리 목록

카테고리는 순서대로 진행하되, AI가 학습자의 상태를 보고 병행하거나 순서를 조정할 수 있다.

<!-- §10-CATEGORY-TABLE -->
| # | 카테고리 | 설명 | 상태 | 시작일 | 완료일 |
|---|---|---|---|---|---|
| A | 서바이벌 표현 | 못 알아들었을 때, 다시 말해달라, 천천히, 잠깐만 등 | 대기 | — | — |
| B | 연음/축약 규칙 | gonna, wanna, didja, shoulda 등 + 강세/리듬 | 대기 | — | — |
| C | 일상 대화 패턴 | 인사, 안부, 날씨, 감사, 사과, 부탁 등 기본 소셜 | 대기 | — | — |
| D | 내 사정 설명하기 | 이유 말하기, 오해 풀기, 배경 설명, 거절하기 | 대기 | — | — |
| E | 여행 실전 | 공항, 호텔, 식당, 택시, 쇼핑, 돌발 상황 | 대기 | — | — |
| F | 넷플릭스 빈출 표현 | 미드/영화에서 자주 나오는 구어체, 슬랭, 관용구 | 대기 | — | — |
| G | 영어 메일 쓰기 | 업무용 기본 이메일 패턴 | 대기 | — | — |
| H | 뉴스/유튜브 이해 | 시사, 테크, 일상 유튜브 청취 | 대기 | — | — |

**상태 데이터:** `english-data.json` → `categories` 배열. 위 테이블은 참조용이며, 실제 상태는 JSON에서 관리한다.

<!-- §10-CATEGORY-RULES -->
### 진행 규칙
- 한 카테고리에서 핵심 표현 30개 이상이 최소 7일 주기(3회차 복습)를 통과하면 "완료"로 표시한다. 30개는 최소 기준이며, 카테고리 범위가 넓은 경우(F, H 등) AI 판단에 따라 완료 기준을 늘릴 수 있다. 기준 변경 시 학습자에게 고지한다.
- 완료된 카테고리의 문장도 60일 복습 주기까지 계속 복습 대상에 포함한다.
- 카테고리 A, B는 병행 진행한다 (서바이벌 + 연음은 모든 단계의 기반).

## 14. AI 행동 규칙

<!-- §14-READ-ORDER -->
### 문서 읽기 우선순위
1. §8-A(복습 큐) — 오늘 복습 대상 파악
2. §6(약점) — 신규 학습에 반영할 약점
3. §10(카테고리) — 현재 진행 중인 카테고리
4. EN-method.md §1(프로필) — 수준 확인 (첫 세션/점검 시)

<!-- §14-GENERATION-RULES -->
### 학습 생성 규칙
- §8-A 복습 큐 전체 행 수가 80을 넘으면 신규 학습을 3개로 제한한다 (아래 규칙보다 우선).
- 복습 대상이 10개 이상이면 신규 학습을 5개로 줄인다.
- 복습 대상이 5개 미만이면 신규 학습을 10개까지 늘린다.
- 전날 약점에 기록된 항목이 있으면 관련 문장을 신규에 1~2개 포함한다.
- 학습자가 "오늘 가볍게"라고 하면 복습만 진행한다.
- 학습자가 "오늘 빡세게"라고 하면 신규를 15개까지 늘린다.
- 학습자가 "넘어가자", "다음"이라고 하면 즉시 다음 단계로 이동한다.

<!-- §14-DIFFICULTY -->
### 난이도 조절 규칙
- 복습 성공률이 최근 3세션 연속 90% 이상이면 다음 카테고리를 병행 시작한다.
- 복습 성공률이 최근 3세션 연속 50% 미만이면 신규 학습을 중단하고 복습에 집중한다.
- 영작에서 같은 실수가 3회 반복되면 해당 문법/패턴을 별도 미니 레슨으로 진행한다.
- "연속"은 날짜가 아니라 세션 기준이다. 이틀 쉬고 3회 학습해도 3세션 연속으로 카운트한다.

<!-- §14-UPDATE -->
### 문서 업데이트 규칙
- 매 학습 종료 시 §15 형식의 작업지시서를 생성한다.
- 약점/강점은 변화가 있을 때만 업데이트한다.
- 학습 통계는 매 세션 종료 시 갱신한다.

<!-- §14-LOG-MANAGE -->
### 학습 로그 용량 관리
- 학습 로그가 30일분을 넘으면 가장 오래된 것부터 요약으로 압축한다.
- 압축 대상: 복습 완료(60일 주기 통과)된 문장의 상세 기록.
- 보존 대상: 현재 복습 주기 안에 있는 문장은 절대 삭제하지 않는다.
- 압축 형식: `Day 1~10 요약: 문장 N개 학습, N개 복습 완료, 주요 약점: [목록]`

<!-- §14-METHODOLOGY -->
### 방법론 점검
- 10일마다 AI가 현재 학습 방법의 효과를 자체 점검한다.
- 복습 성공률, 영작 정확도, 약점 해결 속도를 기준으로 판단한다.
- 비효율적인 부분이 있으면 학습자에게 방법 변경을 제안한다.
- 변경은 학습자가 승인한 뒤에만 적용한다.

<!-- §14-ABORT -->
### 세션 중단 처리
- 학습자가 "오늘 여기까지" 또는 중단 의사를 밝히면, 완료된 Step까지만 학습 로그에 기록한다.
- 복습만 진행하고 신규를 안 한 날도 Day 카운트에 포함한다 (연속 학습 유지).
- Step 2에서 문장 5개 중 3개만 진행하고 끊으면, 나머지 2개는 기록하지 않는다. 다음 세션에서 새로 생성한다.
- 작업지시서는 실제 완료된 내용만 반영한다.

## 15. 문서 업데이트 작업지시서 (VS Agent / Haiku용)

### 역할 분담

| 역할 | 담당 | 원칙 |
|---|---|---|
| 학습 진행 + 판단 | Opus 4.6 | 복습 판정, 신규 생성, 약점 분석, 난이도 조절 — **모든 지적 판단** |
| 작업지시서 생성 | Opus 4.6 | 세션 종료 시 판단 결과를 확정값으로 출력 |
| 파일 편집 | Haiku (VS Agent) | 작업지시서를 읽고 JSON을 편집 — **판단하지 않음** |

### 트리거

학습자가 대화 중 **"작업지시서"**라고 말하면, Opus는 즉시 해당 세션의 변경 사항을 반영한 작업지시서를 출력한다.

### 대상 파일

`english-data.json` (Google Drive, GAS 경유)

EN-core.md / EN-method.md는 방법론 변경 시에만 수정한다. 매 세션 작업지시서에서는 JSON만 다룬다.

→ 작업지시서 상세 문법, Haiku 실행 규칙, 출력 예시는 EN-ref.md §15-A를 참조한다.

<!-- §16-MANAGEMENT -->
## 16. 문서의 관리

이 문서의 목적은 AI가 매 세션 읽고 행동 규칙과 데이터 스키마를 파악하는 것이다. 학습 상태 데이터는 `english-data.json`에서 관리한다.

**핵심 원칙:** EN-core.md가 행동 규칙을, EN-method.md가 방법론을, english-data.json이 학습 상태를 각각 담당한다.

**EN-core.md 갱신 시점:** §14 행동 규칙이나 데이터 스키마가 변경될 때.
**EN-method.md 갱신 시점:** 방법론이 변경될 때. 매 세션 갱신 불필요.
**english-data.json 갱신 시점:** 매 학습 세션 종료 시. §15의 작업지시서 형식을 따른다.

**일본어 학습과의 관계:** JA-core.md / JA-method.md / japanese-data.json과 독립적으로 운영한다. 양쪽 학습을 같은 날 해도 각 JSON은 별도로 업데이트한다. 다만 §14의 방법론 점검 시 양쪽의 학습 부담이 과하지 않은지 AI가 총량을 확인한다.
